# Stage 2: Story Spine (7-Point Structure)

Expand the premise into a 7-point story structure.

## Requirements

**Length:** 500–800 words total (2-3 paragraphs per point)

**Must include:**

1. **Hook** — The inciting incident. What disrupts the protagonist's world?
2. **Plot Turn 1** — First major change. What new direction does the protagonist take?
3. **Pinch Point 1** — First pressure. What forces push against the protagonist?
4. **Midpoint** — Major twist. What changes everything?
5. **Pinch Point 2** — Second pressure. What makes it seem impossible?
6. **Plot Turn 2** — Final turn. What tips it toward climax?
7. **Resolution** — How does it end? What's changed?

## Format

```
# Story Spine

## 1. Hook
[2-3 paragraphs]

## 2. Plot Turn 1
[2-3 paragraphs]

## 3. Pinch Point 1
[2-3 paragraphs]

## 4. Midpoint
[2-3 paragraphs]

## 5. Pinch Point 2
[2-3 paragraphs]

## 6. Plot Turn 2
[2-3 paragraphs]

## 7. Resolution
[2-3 paragraphs]
```

## Constraints

- Each point: 2-3 paragraphs MAX
- The midpoint (point 4) should be a true twist, not just escalation
- Resolution should answer the theme, not just the plot
- Stay true to the tone in premise

## Memory Anchor

Before writing, read `premise.md`.

## Output

Save as `story_spine.md` in project root.
