## Book Factory Fix - March 3, 2026

### The Problem
- Ollama CLI (`ollama run`) outputs VT100/ANSI escape codes in non-TTY mode
- These codes corrupt chapter files - 13 chapters destroyed
- Could not restore from git (committed while corrupted)

### Root Cause
- `ollama run` behaves differently without a terminal - outputs spinner/progress codes
- These codes look like: `[?2026h[?25l⠋ [K[1G etc.

### Solution Found
- Use Python requests to call Ollama API directly instead of CLI
- API endpoint: http://localhost:11434/api/generate
- Returns clean JSON response without ANSI codes
- Config: {"model": "llama3.2", "prompt": ..., "stream": false, "options": {"num_ctx": 8192, "num_predict": 4000}}
